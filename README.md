# 📖 项目简介

**校声（Echo Campus）** 是一款以移动端为核心的校园社交平台原型。

它并不试图复刻传统贴吧、论坛或 QQ 群，而是尝试结合：

* 小红书的信息流体验
* 微博的话题趋势机制
* Discord 的兴趣圈子结构
* 半匿名校园树洞文化

构建一种：

> **更轻松、更自由、更符合当代大学生表达习惯的校园社交空间。**

在这里：

* 用户可以基于兴趣而非班级关系建立连接
* 可以公开表达，也可以匿名倾诉
* 可以刷推荐流，也可以追踪校园热点
* 可以自由控制自己的隐私边界
* 可以在圈子与话题中形成长期社区关系

整个系统围绕：

* 内容推荐
* 轻社交
* 隐私控制
* 趋势发现
* 兴趣连接

进行设计。

# ✨ 核心特色

## 1. 多维信息流系统

首页并非单一推荐流，而是由：

* 遇见
* 关注
* 推荐
* 同城
* 本校

五大内容频道组成。

系统支持：

* 热门 / 最新排序
* 标签筛选
* 时间范围过滤
* 常看频道排序
* 推荐频道动态学习

用户既可以“无目的刷帖”，
也可以精准筛选自己真正关心的内容。

---

## 2. 热点趋势发现机制

发现页采用类似微博热搜的话题结构：

* 热门趋势
* 实时上升
* 本校热议
* 同城热点
* 兴趣话题

用户点击话题后：

不仅可以浏览相关帖子，
还可以：

* 直接参与话题讨论
* 查看相关圈子
* 进入趋势详情页互动

而不是只能“看帖子”。

---

## 3. 圈子系统（Community）

平台内置完整的兴趣圈子体系：

* 摄影
* 音乐
* 游戏
* 考研
* 二手
* 校园活动
* 动漫
* AI / 编程

等兴趣社区。

圈子广场支持：

* 推荐排序
* 热度排序
* 分类浏览
* 搜索加入

用户可以在圈子内形成长期社交关系，
而不是一次性的信息流互动。

---

## 4. 隐私与可见性体系

项目重点设计了：

# “可控边界感”

用户可以单独控制：

* 我的帖子
* 我的评论
* 我的收藏
* 我的圈子

的可见范围：

* 所有人可见
* 仅关注者可见
* 仅互关可见
* 仅自己可见

即使不可见，
访问者仍能看到栏目存在，
但会显示锁定状态。

这种设计避免了：

“主页突然缺一块”的割裂感。

---

## 5. 游客访问限制机制

未登录用户：

可以浏览：

* 首页推荐
* 热门趋势
* 帖子详情
* 话题详情

但无法：

* 点赞
* 收藏
* 评论
* 私信
* 关注
* 发布内容

系统会在对应场景下：

以轻提示 Dialog 的方式引导登录，
而不是直接粗暴跳转。

---

## 6. 长按交互与内容反馈

信息流支持：

* 长按帖子
* 长按聊天
* 长按联系人

呼出操作菜单。

例如：

### 对帖子：

* 不感兴趣
* 减少此类内容

系统会动态降低：

* 标签权重
* 作者权重
* 圈子权重

实现轻量级推荐学习。

# 🎨 设计风格

项目整体采用：

## Mobile First + Glassmorphism

设计思路。

核心特点：

* 拟真手机壳容器
* 毛玻璃磨砂效果
* 低饱和浅绿色主题
* 极细分割线布局
* 去重卡片化设计
* 右手拇指优先交互
* 信息流沉浸式排版

目标是：

在保证信息密度的同时，
尽可能减少视觉压迫感。
